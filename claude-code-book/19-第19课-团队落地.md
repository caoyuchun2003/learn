# 第19课 · 团队落地

## 本课目标

- 制定团队级 Claude Code 使用规范
- 明确人机分工

## 19.1 团队清单

- [ ] 根目录统一 `CLAUDE.md`
- [ ] 测试命令写死在 CLAUDE.md
- [ ] 禁止 AI 改生产配置
- [ ] PR 必须人审 diff
- [ ] 共享 Skills / 常用 Commands
- [ ] 敏感仓库禁用或限制 MCP

## 19.2 人机分工

| 人做 | AI 做 |
|------|-------|
| 定需求、审方案 | 探索代码、写初稿 |
| 审 diff、merge | 跑测试、写 commit 草稿 |
| 架构决策 | 重复性实现 |
| 安全与合规 | 文档、注释 |

## 19.3 新人 Onboarding

1. 装 Claude Code
2. 读团队 CLAUDE.md
3. 用 `/onboard`（如有）做环境检查
4. 从小任务开始（修 typo → 小功能）

## 本课练习

为你团队写 5 条「AI 使用红线」。

## 本课小结

> 团队用得好的关键不是工具，是规范 + 验收文化。
