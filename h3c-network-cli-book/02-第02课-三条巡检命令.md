# 第 02 课 · 三条巡检命令

> 上机验收固定顺序：接口 → 路由 → 配置。
>
> **作者：曹宇春**

---

## 先记住这个类比

巡检像 **医生三板斧**：

| 命令 | 查什么 | 类比 |
|------|--------|------|
| `dis int b` | 口 Up/Down | 血管通不通 |
| `dis ip routing-table` | 路由往哪走 | 导航对不对 |
| `dis cur` | 当前配置 | 病历本 |

**任何验收、排障，先跑这三条。**

---

## 1. display interface brief

```text
<H3C> display interface brief
```

关注：

- **Link** / **Protocol** 是否 UP  
- 错包、流量是否异常  
- 哪个口 Down 了  

简写：

```text
dis int b
```

---

## 2. display ip routing-table

```text
<H3C> display ip routing-table
```

关注：

- 有没有 **0.0.0.0/0** 默认路由  
- 到目标网段的 **下一跳** 对不对  
- 是 static / direct / ospf / bgp  

简写：

```text
dis ip rout
```

---

## 3. display current-configuration

```text
<H3C> display current-configuration
```

关注：

- vlan、interface、ip address、route、nat、acl 是否与方案一致  

简写：

```text
dis cur
```

**改配置前**：复制 `dis cur` 到本地记事本备份。

---

## 验收标准流程（checklist）

```
□ display version          → 型号、Comware 版本
□ display interface brief  → 关键口 UP
□ display ip interface brief → IP 配对了没
□ display ip routing-table   → 默认路由/明细路由
□ ping 网关 / ping 对端
□ display current-configuration → 与方案核对
□ 已 save
```

---

## 常用补充 display

| 命令 | 何时用 |
|------|--------|
| `dis version` | 确认版本 |
| `dis device` | 框式设备板卡 |
| `dis vlan` | 交换 VLAN |
| `dis arp` | IP 和 MAC 对应 |
| `dis cpu-usage` | 卡顿时 |
| `dis logbuffer` | 看最近日志 |

## 练习

1. 口述：口 Up 但 ping 不通网关，接下来 dis 哪两个？  
2. 为什么改生产前要先 `dis cur` 备份？  

---

## 小结

- 巡检顺序：**int b → routing-table → cur**  
- 再加 ping 验证  
- 改前备份 cur，改后 save
