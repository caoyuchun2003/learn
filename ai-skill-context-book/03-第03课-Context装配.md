# 第03课 · Context 装配

## 本课目标

- 知道 Context 包含什么
- 会按场景装配 Context 包

## 3.1 Context 里有什么？

- 对话历史
- `@` 引用的文件
- 报错、日志、截图描述
- `CLAUDE.md`、README
- RAG / MCP 查回的资料
- 当前 git diff

> AI 只看得见你给的 Context，看不见你的屏幕和脑子里的假设。

## 3.2 常备 Context 包

| 层级 | 放什么 |
|------|--------|
| 项目级 | CLAUDE.md、README、测试命令 |
| 任务级 | `@` 相关文件、完整报错 |
| 环境级 | 语言/框架版本、分支名 |
| 约束级 | 「只改 xxx」「不要 xxx」 |

## 3.3 常见问题

| 现象 | 对策 |
|------|------|
| 改错文件 | `@` 精确指向 + 说范围 |
| 忘了前面说的 | 新开对话 + CLAUDE.md 补背景 |
| 用过时 API | MCP 查最新文档 |
| 瞎猜结构 | 先让它探索再动手 |

## 3.4 Context 与提示工程

提示工程 ≈ **组装 Context 的艺术**——不是背咒语，是把该给的信息给全、给准。

## 本课练习

选一个任务，按四层 Context 包各写一条具体内容。

## 本课小结

> Context 装配对了，同样的 Skill 效果差十倍。
