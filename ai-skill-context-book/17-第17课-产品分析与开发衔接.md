# 第17课 · 产品分析与开发衔接

## 本课目标

- 分析结论变成可开发的 Context
- 技术粗评进 PRD

## 17.1 衔接链路

```
产品分析输出
    ↓
PRD / 用户故事 + 验收标准 + 不做范围
    ↓
CLAUDE.md 或任务描述
    ↓
Claude Code 开发 + 测试 Feedback
```

## 17.2 分析 → 开发要传递什么？

| 分析产出 | 开发 Context |
|----------|----------------|
| MVP 3 功能 | 任务范围 |
| 不做清单 | 「不要 xxx」约束 |
| 验收标准 | 测试用例 / Feedback |
| 关键假设 | 先做原型验证再全量开发 |

## 17.3 技术粗评 Skill（开发者专用）

在产品分析末尾加一节：

| 维度 | 问题 |
|------|------|
| 实现路径 | MVP 动哪些模块？ |
| 依赖 | API、模型、合规 |
| 风险 | 隐私、成本、幻觉 |
| 验证 | 先假按钮还是直接开发？ |

## 开口模板

```
（完成产品分析后）
加一节「技术粗评」：MVP 实现路径、主要风险、
建议技术验证项。不写具体代码。
```

## 17.4 开发开口模板（承接分析）

```
【背景】（粘贴 PRD 摘要或第 7 节 MVP）
【验收】（粘贴验收标准）
【约束】（粘贴不做清单）
【参考】@相关模块

按 MVP 实现第 1 项，最小改动，做完跑测试。
```

## 本课练习

拿第 15 课练习的输出，写 5 条「开发约束」和 3 条「验收标准」。

## 本课小结

> 好的产品分析，结尾就能直接复制进 Claude Code 的任务框。
