# 第 19 课 · 安全威胁与攻击面

> 协议层、实现层、应用层分别怕什么。
>
> **作者：曹宇春**

---

## 协议层威胁

| 威胁 | 说明 | 缓解 |
|------|------|------|
| 51% 攻击 | 算力 majority 重组 | 多确认、算力去中心化 |
| 双花 | 同一 UTXO 花两次 | 全节点拒绝；商户多确认 |
| 芬尼攻击 | 0-conf 商户即时交付 | 不接受未确认大额 |
| 时间扭曲 | 操纵 timestamp 影响难度 | 共识限制时间偏差 |

---

## 钱包 / 应用层威胁

| 威胁 | 说明 | 缓解 |
|------|------|------|
| 助记词泄露 | 钓鱼、木马、云备份 | 硬件钱包、离线签名 |
| 地址替换 | 剪贴板恶意软件 | 硬件屏确认、地址校验 |
| 假钱包 App | 偷词、偷私钥 | 官方渠道、开源可审计 |
| 交易所被黑 | 托管风险 | 自持密钥 |
| 供应链攻击 | 依赖库后门 | 校验 checksum、复现构建 |

---

## 开发层陷阱

| 陷阱 | 后果 |
|------|------|
| 错误 SIGHASH | 签名可被复用到别的交易 |
| 整数溢出（历史） | 比特币已用 satoshi 整数 |
| 重用 nonce（ECDSA） | 可推私钥（2010 PlayStation 类事故） |
| 不验证 output 金额 | 被 fee 攻击 |
| 信任第三方 fee 估算 | 多付或 stuck |

**Schnorr/MuSig** 需注意 nonce 生成必须密码学安全随机（或用 RFC6979 类确定性方案）。

---

## 隐私与链分析

- 地址重用 → 关联所有交易
- KYC 交易所入金 → 链上地址 deanonymize
- CoinJoin / PayJoin 提高隐私（钱包如 Wasabi、Samourai 原理）
- **Taproot** 默认让多签与单签看起来相似

---

## 安全开发清单

```
□ 在 regtest/signet 充分测试
□ 大额用 PSBT + 硬件签名
□ 核对 output 地址与金额（人机双重）
□ 不记录助记词到日志/数据库
□ 多签 + 地理分散备份
□ 监控异常出块 / 深度重组（交易所）
```

---

## 本课小结

- 比特币协议久经考验；风险多在**密钥管理与应用**
- 0-conf 不适合高价值交付
- 开发者对签名语义和金额校验负全责
