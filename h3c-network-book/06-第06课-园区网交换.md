# 第 06 课 · 园区网交换

> 接入—汇聚—核心：S5130、S5560、S7500 各放哪一层？
>
> **作者：曹宇春**

---

## 先记住这个类比

园区网 = **一栋（或一片）办公城**：

| 层级 | 类比 | 华三典型系列 |
|------|------|--------------|
| 接入 | 工位 **网线口** | S5120/S5130、S5560 |
| 汇聚 | 楼层 **弱电间** | S5560、S6520 |
| 核心 | **园区主干道枢纽** | S7500/S7500E、S10500 |

```
        S7500 IRF 核心
       /      |      \
   汇聚      汇聚      汇聚
    |         |         |
  S5130     S5130     S5130  (+ PoE → AP)
```

---

## 接入层：S5130 是主力

| 特性 | 说明 |
|------|------|
| 端口 | 全千兆下行 + 万兆上联 |
| PoE+ | 给 AP、IP 电话、摄像头供电 |
| 认证 | 802.1X、Portal，配合 iMC |

**PoE 预算**：先算每个 AP 要 30W 还是 60W，再选交换机总功率——**无线项目最常见翻车点**。

---

## 核心层：IRF 双核心模板

```
   [ S7500E-A ]══IRF══[ S7500E-B ]
            │
      双 uplink LAG
            │
         汇聚层
```

**为什么 IRF 而不是两台独立核心？** 逻辑一台、配置一份、跨框 LAG，STP 简单。

---

## Cloudnet：中小客户极简方案

无专职网管 → **Cloudnet 云管** + 支持云管的 S5560 + MSR810：  
手机 App 开局，适合 **连锁门店、小微分支**。

---

## 典型行业

| 行业 | 组合要点 |
|------|----------|
| 高校 | S10500 核心 + 海量 S5130 + 高密 AP |
| 医院 | IRF 冗余 + 业务 VLAN 严格隔离 |
| 企业总部 | S7500E + MSR/SR 出口 + 独立 AC |

---

## 在你项目里什么时候用得上

| 现象 | 查什么 |
|------|--------|
| 信息点千兆够用 | S5130 |
| 核心要 40G/100G 上联 | S7500E / S10500 |
| 只要几十口、要云管 | Cloudnet + S5560 |
| Wi-Fi 6 三射频 AP | PoE++ 交换机 |

## 小结

- 园区 = 接入 S5130 / 汇聚 S5560–S6520 / 核心 S7500–S10500  
- IRF 双核心是华三园区高可用标配  
- PoE 预算与 AC 容量要一起算
