## 📌 导读（全景总结）

### 一切的地基：三句话

> 全书长在这三个根上，所有「该上什么型号」都能追到这里。

```
① 网络基础 = 路由（连外部）+ 交换（连内部）+ 无线（连移动）+ 网管（连运维）
② Comware = 统一操作系统：学会一块，其他设备命令体系相通
③ 场景决定型号：分支用 MSR，园区用 S5130/S7500，数据中心用 S6800/S12500
```

### 全书核心类比（反复使用）

| 技术概念 | 生活类比 |
|----------|----------|
| 企业网整体 | 一座「园区城市」 |
| MSR 路由器 | 分支 **快递站**：收发外部包裹、VPN 专线 |
| SR 路由器 | 城市 **高速枢纽/省界收费站** |
| S 系列接入交换 | 楼层 **内线电话/工位配线架** |
| S 系列核心交换 | 园区 **交通主干道枢纽** |
| S6800/S12500（DC） | 数据中心 **货架通道 + 立体仓库传送带** |
| WA AP | 园区 **Wi-Fi 热点** |
| AC 无线控制器 | **统一管所有热点的调度中心** |
| iMC 网管 | **交通监控大屏 + 报警中心** |
| AD-NET / AD-WAN | **智能调度系统**（自动配路、选最优链路） |
| Comware | 所有设备的 **统一「操作系统」** |
| IRF 堆叠 | 两台核心交换机 **合体成一个逻辑大脑** |
| VXLAN/EVPN | 数据中心 **虚拟邮编系统**（租户隔离） |

### 五条核心心法（只记这 5 句）

1. **先分场景，再选型号**——分支、园区、DC 是三条产品线，别用 S5130 当 DC Leaf
2. **MSR 管「外面」，S 系列管「里面」**——出口 VPN 找 MSR，工位接入找 S5130
3. **核心靠 IRF，DC 靠 Spine-Leaf**——园区高可用堆 IRF；新建 DC 别再用大三层层级
4. **无线别忘 PoE 预算**——AP 要供电，接入交换机 PoE 功率先算再配
5. **网管不是可选项**——设备多了没有 iMC/AD-NET，等于没有监控大屏

## 电梯演讲（30 秒版）

> 华三网络基础就是四件事：MSR/SR 路由器连分支和核心，S 系列交换机组园区和数据中心，WA+AC 做无线，iMC 和 AD-NET 管运维。它们都跑 Comware 系统。分支一台 MSR 出 VPN，园区 S5130 接入 + S7500 核心堆 IRF，数据中心 S6800 当 Leaf、S12500 当 Spine，上面跑 VXLAN。
