# 第 18 课 · 钱包标准 BIP-32/39/44

> HD 钱包、助记词与派生路径——库实现的基础。
>
> **作者：曹宇春**

---

## BIP-39：助记词

- 128–256 bit 熵 → 映射为 **12/15/18/21/24** 个英文词
- 含校验和，错一个词大概率无法恢复
- **PBKDF2** 从助记词 + 可选 passphrase 导出 seed

```
助记词 → seed (512 bit) → BIP-32 主密钥
```

**passphrase：** 可选第 25 个词，防助记词泄露被直接花掉（需记牢）。

---

## BIP-32：分层确定性密钥

从 seed 导出树状密钥：

```
m / purpose' / coin_type' / account' / change / address_index
```

- `m`：主节点
- `'`：硬化派生（hardened），子公钥不能推兄弟私钥
- **xpub**：扩展公钥，可派生收款地址但不能花（无私钥）

```bash
bitcoin-cli -regtest listdescriptors
# 现代 Bitcoin Core 默认 descriptor 钱包
```

---

## BIP-44 / 49 / 84 / 86：coin type 与脚本类型

| BIP | purpose | 路径示例 | 地址类型 |
|-----|---------|----------|----------|
| 44 | 44' | m/44'/0'/0'/0/0 | P2PKH (1...) |
| 49 | 49' | m/49'/0'/0'/0/0 | P2SH-P2WPKH (3...) |
| 84 | 84' | m/84'/0'/0'/0/0 | P2WPKH (bc1q...) |
| 86 | 86' | m/86'/0'/0'/0/0 | P2TR (bc1p...) |

`0'` = Bitcoin mainnet；testnet 常用 `1'`。

---

## Descriptor 钱包（现代标准）

Bitcoin Core 22+ 默认 **descriptor wallet**，用字符串描述脚本：

```
pkh(...)           # P2PKH
wpkh(...)          # P2WPKH
tr(...)            # Taproot
wsh(sortedmulti(...))  # 多签
```

```bash
bitcoin-cli -regtest listdescriptors true
bitcoin-cli -regtest getdescriptorinfo "wpkh(...)"
```

优于老的 BIP-44 账户模型：表达力强、可导入导出明确。

---

## 实现注意

| 规则 | 原因 |
|------|------|
| 每笔收款用新地址 | 隐私 |
| 离线生成助记词 | 防泄露 |
| 不用脑记词钱包 | 熵不够 |
| 备份 passphrase | 丢了 seed 变砖 |

---

## 本课小结

- BIP-39 → 人类可读 seed
- BIP-32 → 一棵树派生无数地址
- BIP-84/86 → 现代 bc1q / bc1p 路径
- 生产用 descriptor + PSBT
