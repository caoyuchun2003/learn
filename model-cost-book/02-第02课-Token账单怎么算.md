# 第 02 课 · Token 账单怎么算

> 结论：**单价只是分子；Token 量与调用次数才是爆炸点。**

---

## 粗算公式（够用就行）

```
单次成本 ≈ (输入 Token × 输入单价) + (输出 Token × 输出单价)
日成本 ≈ 单次成本 × 日调用次数 × 重试系数
```

重试系数：超时重试 1 次，粗算 ×1.2～1.5（看你的失败率）。

中文大约 **1 汉字 ≈ 1～2 Token**（因分词而异）。估算时偏保守没坏处。

## 真正吃 Token 的地方

| 项 | 说明 |
|----|------|
| System 提示 | 每次都带 |
| 检索摘录 / RAG 片段 | 一塞 Top5 就很长 |
| 多轮 history | 越聊越贵 |
| 模型「思考」过程 | 部分推理模型输出里还含水 |

手册助教把 **history 限制在最近几轮**，就是为了压输入。

## 控制台要对的数

- 输入 / 输出分开计价时，**长答案**和**长摘录**是两头羊  
- 流式不省钱：一般仍按最终 Token 计（看厂商细则）  
- 「测了 10 次感觉不贵」≠「日活千人」

## 一张给产品的估算表（模板）

| 场景 | 估计输入 Token | 估计输出 | 日调用 | 备注 |
|------|----------------|----------|--------|------|
| 手册助教一问 | 1500 | 400 | 100 | 含摘录 |
| … | | | | |

先填表，再选模型。顺序别反。

## 练习

用你现在的 Key 控制台，看最近 7 天 Token 分布：哪一个接口最贵？是输入还是输出？
