## 高层 API（钱包已管理密钥）

```bash
# regtest 环境，已有余额

# 1. 新建收款地址
DEST=$(bitcoin-cli -regtest getnewaddress)

# 2. 发送（内部完成选币、构造、签名、广播）
TXID=$(bitcoin-cli -regtest sendtoaddress $DEST 0.01)
echo $TXID

# 3. 出块确认
bitcoin-cli -regtest generatetoaddress 1 $(bitcoin-cli -regtest getnewaddress)

# 4. 查看交易
bitcoin-cli -regtest gettransaction $TXID
```

## 底层流程（理解协议）

```
1. listunspent     → 选 input UTXO
2. createrawtransaction → 指定 inputs + outputs
3. fundrawtransaction   → 自动选币、加找零、算 fee（可选）
4. signrawtransactionwithwallet / withkey
5. testmempoolaccept    → 预检
6. sendrawtransaction   → 广播
```

### 示例

```bash
# 假设已有 UTXO
UTXO=$(bitcoin-cli -regtest listunspent 1 | jq '.[0]')
TXID=$(echo $UTXO | jq -r '.txid')
VOUT=$(echo $UTXO | jq -r '.vout')

CHANGE=$(bitcoin-cli -regtest getrawchangeaddress)
DEST=$(bitcoin-cli -regtest getnewaddress)

RAW=$(bitcoin-cli -regtest createrawtransaction \
  "[{\"txid\":\"$TXID\",\"vout\":$VOUT}]" \
  "{\"$DEST\":0.001,\"$CHANGE\":0.0089}")

FUNDED=$(bitcoin-cli -regtest fundrawtransaction $RAW)
SIGNED=$(bitcoin-cli -regtest signrawtransactionwithwallet \
  $(echo $FUNDED | jq -r '.hex'))

bitcoin-cli -regtest sendrawtransaction \
  $(echo $SIGNED | jq -r '.hex')
```

## 签名哈希（SIGHASH）

签名不只签整个交易，通过 **SIGHASH 标志**指定签哪些 input/output：

| 标志 | 含义 |
|------|------|
| SIGHASH_ALL | 默认，签所有 input/output |
| SIGHASH_NONE | 不承诺 output |
| SIGHASH_SINGLE | 只承诺同 index 的 output |

开发一般只用 `SIGHASH_ALL`。

## 库构造（JavaScript 概念）

```javascript
// bitcoinjs-lib 伪代码
import * as bitcoin from 'bitcoinjs-lib'

const psbt = new bitcoin.Psbt({ network: bitcoin.networks.regtest })
psbt.addInput({ hash: txid, index: vout, witnessUtxo: { script, value } })
psbt.addOutput({ address, value })
psbt.signInput(0, keyPair)
psbt.finalizeAllInputs()
const tx = psbt.extractTransaction()
```

现代流程优先 **PSBT**（下一课），而非裸 `createrawtransaction`。
