## 先记住分工

| 工具 | 干什么 | 是不是 box CLI |
|------|--------|----------------|
| **Comware CLI** | 单台精配、救急、Demo | 是 |
| **iMC** | 拓扑、告警、备份、批量下发 | 服务器 Web |
| **AD-NET** | SDN、VXLAN 自动化 | 控制器 Web/API |
| **AD-WAN** | 分支 SD-WAN 策略 | 控制器 Web |
| **NETCONF/Ansible** | 脚本批量、GitOps | API |

---

## 何时必须上自动化

| 规模 | 建议 |
|------|------|
| < 10 台 | CLI + iMC 可选 |
| 10–100 台 | iMC + 配置模板 |
| 100+ 台 / 多分支 | AD-WAN、Ansible |
| DC Leaf 数十台+ | AD-NET、NETCONF |

---

## NETCONF 概念

Comware 支持 **NETCONF + YANG**：

- 结构化 XML/JSON 配置  
- Ansible `h3c_comware` 等模块可批量改 VLAN、接口  
- 适合 **重复性变更**（开 100 个端口 access vlan）

示例思路（非完整 playbook）：

```yaml
# Ansible 思路：对 group switches 批量配 VLAN10
- name: Ensure VLAN 10
  comware_vlan:
    vlanid: 10
    name: Office
```
