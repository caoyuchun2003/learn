# 第 02 课 · Comware 与网络架构演进

> 华三网络的「统一灵魂」：从 VLAN 到 VXLAN，从 IRF 到 AD-NET。
>
> **作者：曹宇春**

---

## 先记住这个类比

**Comware** = 所有华三网络设备的 **同一套「城市管理条例」**：

- 在 MSR 上写 VPN 规则  
- 在 S7500 上配 IRF 堆叠  
- 在 S12500 上跑 VXLAN  

命令风格、特性名称 **大体相通**——学会一块，迁移成本低。

| 技术 | 类比 |
|------|------|
| **IRF** | 两台核心交换机 **合体成一个逻辑大脑** |
| **VXLAN/EVPN** | 数据中心 **虚拟邮编**（租户隔离，突破 4096 个 VLAN） |
| **AD-NET** | **智能交通调度**（SDN：业务意图自动变成配置） |
| **AD-WAN** | 分支 **多链路智能选路**（哪条快走哪条） |

## IRF：园区核心的标配语言

**IRF（Intelligent Resilient Framework）** 把多台物理交换机虚拟成 **一台逻辑设备**：

```
   [ 框 A ]════IRF════[ 框 B ]   ← 逻辑上一台核心
            │
        汇聚层（双 uplink LAG）
```

**好处**：配置一份、跨框链路聚合、单框故障秒级切换。  
**限制**：需同系列兼容型号，堆叠带宽受物理连线限制。

---

## 架构演进：三层 → Spine-Leaf

**园区（today 仍常见）**

```
核心（S7500 IRF）→ 汇聚（S5560）→ 接入（S5130 + PoE）
```

**数据中心（新建主流）**

```
Spine（S12500）─── Leaf（S6800）─── 服务器
         └── EVPN/VXLAN overlay ──┘
```

旧 DC「核心-汇聚-接入 + STP」在大二层、VM 迁移场景 **扩展性差**；Spine-Leaf + VXLAN 是 **2010 年代后的标准答案**（第 07 课详述）。

## iMC 与 AD-NET 分工

| 产品 | 干什么 |
|------|--------|
| **iMC** | 拓扑、告警、配置备份、报表（传统网管） |
| **AD-NET** | SDN 策略、VXLAN 自动化、业务随行 |
| **AD-WAN** | 分支 SD-WAN，与 MSR 配合（第 10 课） |

---

## 在你项目里什么时候用得上

- 客户要求 **双核心不宕机** → 提 IRF，别只堆两台独立核心 + VRRP  
- 私有云/多租户 DC → 提 VXLAN/EVPN，别硬扩 VLAN  
- 多分支选路优化 → AD-WAN + MSR，不是手工静态路由

## 小结

- Comware 统一路由、交换、无线 AC 的软件能力  
- IRF = 园区核心冗余；VXLAN/EVPN = DC overlay  
- iMC 管设备健康，AD-NET/AD-WAN 管策略自动化
