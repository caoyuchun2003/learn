# 附录 · 个人 Skill 库模板

复制到你的笔记或 `CLAUDE.md` 旁，持续迭代。

---

## 1. 调试 Skill

```
现象 → 假设 → 验证 → 最小修复
先分析，不瞎改；改完跑测试贴结果。
```

## 2. 小功能 Skill

```
参考现有写法；最小改动；只改相关文件；加验证。
```

## 3. Code Review Skill

```
维度：正确性、简洁性、一致性、测试、安全。
只列问题，按严重程度排序，先不改代码。
```

## 4. 安全红线 Skill

```
不碰 .env / 生产配置
不 git push --force
大改前开分支；merge 前人审 diff
```

## 5. 项目规范 Skill

```
（从 CLAUDE.md 同步）
技术栈：
测试命令：
命名规范：
禁止：
```

## 6. 产品分析 Skill

```
输出结构 1-7（见第15课）
事实 / 推断 / 假设 分开
不编造数据；必须有不做清单
```

## 7. 文档 Skill

```
结构：目标 → 正文 → 练习 → 小结 → 页脚署名
```

---

## 维护习惯

- 每次 AI 干得漂亮 → 把 prompt 抽一条进 Skill 库
- 每次翻车 → 补一条 Context 或约束
- 每月清理重复、合并同类
