# 第14课 · 产品分析概论

## 本课目标

- 用 Skill + Context 做产品分析
- 分清 AI 能做什么、不能做什么

## 14.1 产品分析公式

```
产品分析效果 = 分析框架（Skill）× 真实材料（Context）× 可验证结论（Feedback）
```

## 14.2 AI 擅长 / 不擅长

| 擅长 | 不擅长 |
|------|--------|
| 结构化、穷举维度 | 编造用户数、市场规模 |
| 挑漏洞、列假设 | 替你去访谈用户 |
| 竞品对比框架 | 替你做最终拍板 |

## 14.3 必守原则

1. **事实 / 推断 / 假设** 分开写
2. 无材料的数据标 **「待验证」**
3. 必须有 **「不做清单」**
4. 结论要能对应 **下一步动作**

## 14.4 和开发的关系

```
产品分析（想清楚）→ PRD → 开发（CLAUDE.md + 验收标准）
```

分析阶段的「验收标准、不做范围」应进入开发 Context。

## 本课练习

用一句话写你正在想的一个产品，标出它是「事实」还是「假设」。

## 本课小结

> 产品分析不是写漂亮 PPT，是支撑决策 + 列清待验证假设。
