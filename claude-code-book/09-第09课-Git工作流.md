# 第09课 · Git 工作流

## 本课目标

- 让 Claude Code 协助 commit、分支、PR
- 保持 Git 历史清晰

## 9.1 常用场景

| 场景 | 怎么说 |
|------|--------|
| 写 commit message | 根据 staged diff 写 conventional commits |
| 开分支 | 为 xxx 功能创建 feature 分支 |
| PR 描述 | 总结本分支相对 main 的改动 |

## 9.2 推荐话术

```
根据当前 staged 改动，写一条 conventional commits 格式的 message，
先给我看，确认后再 commit。
```

```
对比 main...HEAD 的 diff，写 PR 描述（Summary + Test plan）。
```

## 9.3 注意

- 不让它自动 push，除非你明确要求
- commit message 要人审
- 大功能一个 PR 一个主题

## 本课练习

做完小改动，让它写 commit message 并解释为什么这样写。

## 本课小结

> Git 是安全网。用好它，AI 改坏了也能回滚。
