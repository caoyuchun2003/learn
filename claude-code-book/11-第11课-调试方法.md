# 第11课 · 调试方法

## 本课目标

- 用结构化方式从报错到修复
- 避免 AI 瞎改一通

## 11.1 调试四步

```
现象 → 假设 → 验证 → 最小修复
```

## 11.2 调试话术

```
报错如下：（完整粘贴）
复现：运行 pnpm test auth.test.ts
请先分析 2-3 个可能原因，
再选最可能的一个用最小改动验证。
```

## 11.3 给 AI 什么？

- ✅ 完整报错栈
- ✅ 复现步骤
- ✅ 你试过的方法
- ❌ 只说「坏了」

## 11.4 验证闭环

修完后必须：
1. 跑失败的那个测试
2. 跑全量测试（如合适）
3. 把结果贴出来

## 本课练习

拿一个真实 bug 走完整调试流程。

## 本课小结

> 好调试 = 好上下文 + 小步验证。别让它一次改十个文件。
