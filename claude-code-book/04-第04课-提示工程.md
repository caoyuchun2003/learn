# 第04课 · 提示工程

## 本课目标

- 掌握编程场景 5 大原则
- 能写出让 Claude Code 少犯错的 prompt

## 4.1 五大原则

| # | 原则 | 差 | 好 |
|---|------|----|----|
| 1 | 具体 | 优化代码 | 加 title 模糊搜索，每页 10 条 |
| 2 | 给上下文 | 报错了 | 贴完整报错 + 代码 + 版本 |
| 3 | 给例子 | 按规范写 | 参考 XxxController@index |
| 4 | 定约束 | （无） | 不要改其他文件 |
| 5 | 结构化 | 帮我做登录 | 先列计划，确认后再实现 |

## 4.2 专用句式

```
先分析方案，不要改文件。
按计划执行第 2 步，做完停。
用最小 diff 修复。
改完后运行 pnpm test，贴结果。
```

## 4.3 万能模板

```
【角色】你是 ___ 专家
【任务】做 ___
【上下文】背景 + 报错/文件
【要求】① ___ ② ___
【约束】不要 ___
```

## 本课练习

同一任务分别用「差 prompt」和「好 prompt」试一次，对比结果。

## 本课小结

> 答不好，多半是你没说清，不是它笨。
