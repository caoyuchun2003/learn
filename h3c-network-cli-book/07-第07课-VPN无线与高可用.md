## 一、IPSec VPN（MSR 分支）结构

**不要从零手敲全文**——按总部模板改 IP/密钥/ACL。

### 组件

```
1. ACL（感兴趣流）     哪些网段要进隧道
2. IKE proposal       第一阶段加密算法
3. IKE profile        对端、预共享密钥
4. IPsec policy       第二阶段、绑定 ACL
5. 应用到 WAN 口       ipsec apply policy
```

### 示例骨架

```text
[RT] acl advanced 3100
[RT-acl-ipv4-advanced-3100] rule 5 permit ip source 192.168.10.0 0.0.0.255 destination 10.0.0.0 0.255.255.255
[RT-acl-ipv4-advanced-3100] quit

[RT] ike proposal 10
[RT-ike-proposal-10] quit

[RT] ike profile hq
[RT-ike-profile-hq] keychain hc
[RT-ike-profile-hq] quit

[RT] ipsec policy branch 10 isakmp
[RT-ipsec-policy-isakmp-branch-10] ike-profile hq
[RT-ipsec-policy-isakmp-branch-10] security acl 3100
[RT-ipsec-policy-isakmp-branch-10] remote-address <总部公网IP>
[RT-ipsec-policy-isakmp-branch-10] quit

[RT] interface GigabitEthernet0/0
[RT-GigabitEthernet0/0] ipsec apply policy branch
[RT-GigabitEthernet0/0] quit
```

### 验收

```text
display ike sa
display ipsec sa
ping 10.0.0.1
```

## 二、无线（AC）

### 最常打的三条

```text
display wlan ap all
display wlan client
display wlan ssid
```

AP 状态 **Run** = 正常；**Idle** = 未上线。

### 配置入口

```text
[AC] wlan
[AC-wlan] service-template office
[AC-wlan-service-template-office] ssid Office-WiFi
```

> 无线 CLI **版本差异大**，以 WLAN 配置指南为准；实验优先 **dis wlan ap all**。

### AP 不上线排查链

```
dis wlan ap all → dis int b（PoE 口）→ dis vlan → ping AC
```

## 三、IRF 堆叠（核心交换）

**概念命令**（实际步骤依型号文档）：

```text
display irf
display irf topology
```

配置涉及 `irf member`、`irf-port`、MAD 检测——**必须在官方 IRF 指南指导下操作**，Demo 环境再练。

验收：两台框 `display irf` 显示同一 IRF 域。

## 四、VRRP（双机网关）

```text
[SW] interface Vlan-interface 10
[SW-Vlan-interface10] vrrp vrid 1 virtual-ip 192.168.10.254
[SW-Vlan-interface10] vrrp vrid 1 priority 120
```

验收：`display vrrp`
