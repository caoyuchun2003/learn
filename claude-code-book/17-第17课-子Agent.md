# 第17课 · 子 Agent

## 本课目标

- 学会拆解复杂任务
- 理解探索 / 实现 / 测试分工

## 17.1 何时拆子任务？

- 涉及多个模块
- 需要并行探索
- 实现和审查要分离

## 17.2 推荐分工

| 角色 | 职责 |
|------|------|
| 探索 | 摸清代码结构，不改文件 |
| 实现 | 按方案写代码 |
| 测试 | 补用例、跑 CI |
| 审查 | Review diff |

## 17.3 话术示例

```
这个任务分三步：
1. 先探索 auth 模块边界，出报告，别改代码
2. 实现 refresh token，最小改动
3. 补测试并跑 pnpm test
每步结束汇报，等我确认再下一步。
```

## 本课练习

给一个「加导出 Excel 功能」需求，写出 3 步子 Agent 分工计划。

## 本课小结

> 复杂任务像项目管理：拆清楚，比一次塞给 AI 更靠谱。
