# 第 07 课 · 数据中心交换

> 从 S6800 Leaf 到 S12500G Spine：VXLAN、EVPN 与智算 RoCE。
>
> **作者：曹宇春**

---

## 先记住这个类比

| 角色 | 类比 | 典型系列 |
|------|------|----------|
| **Leaf** | 货架旁的 **传送带接驳口**（连服务器） | S6800、S6900 |
| **Spine** | 仓库 **主通道交叉节点** | S9820、S12500/S12500X |
| **智算 Spine** | **加宽、低延迟的 AGV 主轨** | S12500G-AF |
| **VXLAN** | 每个租户的 **独立虚拟邮编** | Comware V7/V9 |

---

## Spine-Leaf：新建 DC 默认架构

```
   Spine ─── Spine ─── Spine
     \  |  / \  |  /
      \ | /   \ | /
       Leaf   Leaf
        |      |
      服务器  服务器
```

- **任意 Leaf 到 Leaf 固定跳数**（经 Spine）  
- **ECMP** 充分利用链路  
- **Overlay VXLAN** 租户与物理拓扑解耦  

旧式「核心-汇聚-接入 + 大 STP 域」→ **新建少选**。

---

## Leaf：S6800 / S6900

| 能力 | 说明 |
|------|------|
| ToR | 10G/25G/100G 接服务器 |
| VTEP | 硬件 VXLAN 线速 |
| 双 ToR | M-LAG/DRNI + 服务器双网卡 |

---

## Spine：S12500 系列

| 系列 | 定位 |
|------|------|
| S9820 | 中型 DC Spine |
| S12500 / S12500X | 大型 DC |
| S12500G-AF | 智算、400G、低时延 |

关注：**交换容量、100G/400G 密度、Buffer、EVPN 表项规模**。

---

## 智算网络（2020s 重点）

GPU 训练 **AllReduce** 对 **微秒时延、低丢包** 敏感 → **RoCEv2**：

| 层次 | 要求 |
|------|------|
| 物理 | 100G/400G，优质光模块 |
| 流控 | PFC、ECN |
| 拓扑 | 非阻塞 Spine-Leaf |

**通用云 DC** 与 **智算 DC** 差别：后者 **时延/丢包** 指标 far 更严，端口速率 **100G/400G 为主**。

---

## 与 CloudOS 联动

```
CloudOS 开租户 → AD-NET 控制器 → S6800/S12500 下发 EVPN VXLAN
```

**同厂云+网** 是新华三私有云方案的典型交付路径。

## 在你项目里什么时候用得上

| 场景 | 组合 |
|------|------|
| 500 台服务器私有云 | S12500X Spine + S6800 Leaf + EVPN |
| GPU 训练集群 | S12500G-AF + S6900 + RoCE 调优 |
| 别用 S5130 当 ToR | 接入交换 **没有** DC 级 VXLAN 规模 |
