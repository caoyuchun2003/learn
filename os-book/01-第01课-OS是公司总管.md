# 第 01 课 · OS 是公司总管

> 先建立「公司模型」：后面每一课都是这家公司的一个部门。
>
> **作者：曹宇春**

---

## 先记住这个类比

把电脑想成一家公司：

| OS 概念 | 类比 |
|---------|------|
| **硬件** | 公司资产：办公室、电脑、仓库、网线 |
| **操作系统** | 行政总管：排班、分资源、定规矩 |
| **应用程序** | 各部门员工：开发、设计、浏览器各自干活 |
| **你** | 老板：下任务、审批、最后拍板 |

没有总管，员工会抢办公室、抢仓库、互相踩脚。有了 OS，大家按规则协作。

---

## 正式概念（人话版）

**操作系统（OS）** 夹在硬件和应用中间，干五件事：

1. **进程管理**：谁上班、怎么换班  
2. **内存管理**：谁用哪张桌面  
3. **文件系统**：档案怎么存怎么找  
4. **设备管理**：键盘、屏幕、SSD 怎么对接  
5. **权限与接口**：谁能进核心档案室、程序怎么填「领料单」找总管  

macOS 底层是 **Darwin**（类 Unix）。你写业务时用到的 `ps`、`kill`、管道 `|`，和 Linux 服务器 **高度相通**——所以学 Mac 上的 OS，不是浪费，是给部署打基础。

```
应用程序（Chrome、Cursor、微信）
        ↓
   操作系统（macOS / Linux / Windows）
        ↓
   硬件（CPU、内存、SSD、网卡）
```

## 在你电脑上看一眼

```bash
# 系统版本与内核
sw_vers
uname -a

# 机器名、用户（你是老板）
whoami
hostname
```

---

## 开发里什么时候用得上

- 应用「崩了 / 卡了 / 删不干净」——责任常常在 **OS 如何管理进程与文件**，不只在业务代码  
- 上服务器之前：同一套总管思维（进程、端口、权限）直接复用  
- 解释「为什么 IDE 吃很多内存」：总管允许每个部门占自己的桌面  

---

## 练习

1. 用一句话跟同事解释：OS 和「应用」谁管资源谁干活？  
2. 打开「关于本机」，对照：芯片（工位类型）、内存（桌面大小）、存储（仓库容量）。  

---

## 小结

- OS = 行政总管，不是某一个 App  
- 五大职能：进程、内存、文件、设备、权限接口  
- Mac ≈ Unix 系：Terminal 习惯将来可迁到 Linux
