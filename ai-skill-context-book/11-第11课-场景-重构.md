# 第11课 · 场景作战卡：重构

## 本课目标

- 小步重构，测试始终绿
- 不夹带新功能

## Skill

- **小步**：一步一改，每步可验证
- 不顺便加功能、不升级无关依赖
- 没说「重构」就不要大动结构

## Context

- 要重构的文件 `@`
- 对应测试 `@`
- 明确「不要动」的范围

## Feedback

- 每步后相关/全量测试通过
- diff 只有重构意图内的变更

## 开口模板

```
重构 @utils/date.ts：提取重复解析逻辑为 parseDate()，
保持对外 API 不变。每步跑 pnpm test utils。
不要改其他文件。
```

## 红线

| 不要 | 要 |
|------|-----|
| 重构 + 新功能一起做 | 分开两轮 |
| 无测试硬改 | 先补测试再改 |

## 本课练习

列一个你想重构的点，写清「不变 API」和测试命令。

## 本课小结

> 重构的 Feedback 是测试绿灯，不是「看起来更好」。
