## 📌 导读（全景总结）

### 一切的地基：三句话

> 全书长在这三个根上，所有「为什么」都能追到这里。

```
① 操作系统 = 公司的行政总管：分 CPU、内存、磁盘，定规矩、防乱跑
② 物理资源永远稀缺：桌面（内存）和仓库（磁盘）都有限，不够就会「跑仓库」变慢
③ 卡顿 ≈ 调度失败的可见结果：不是芯片一定不行，常常是内存/磁盘/并发抢资源
```

### 全书分三篇

```
地基篇：总管、部门、桌面、前台（第 01–04 课）
现场篇：档案室、寄信、隔间（第 05–07 课）
进阶篇：程序怎么上班、门禁、多人改白板（第 08–10 课）
```

### 全书核心类比（反复使用）

| 技术概念 | 生活类比 |
|----------|----------|
| 操作系统（OS） | 公司行政总管 |
| CPU / 多核 | 前台工位（几个人能同时办事） |
| 进程 | 独立部门 |
| 线程 | 部门里的员工 |
| 物理内存（RAM） | 办公桌面 |
| Swap / 硬盘交换 | 临时塞进仓库，用时再搬回 |
| 内存压缩 | 真空袋（比跑仓库快，仍是桌面不够的信号） |
| 文件系统 | 档案室与目录卡（inode） |
| 沙盒 / Containers | 部门私人隔间 |
| Socket / 端口 | 公司地址 + 分机号 |
| 虚拟机 | 楼内再建一整栋分公司 |
| 容器 | 标准化实验隔间（共享同一套总管） |
| 系统调用 | 员工填领料单找总管 |
| 锁 / 死锁 | 拿令牌改白板；互相等对方先让 |

### 五条核心心法（只记这 5 句）

1. **先看资源，再骂芯片**——Mac 很卡，常是内存/磁盘满了，不是 M 芯片突然废了
2. **进程多 ≠ CPU 忙**——很多人在岗，大半在等档案室（I/O），真抢前台的可能不多
3. **桌面比仓库金贵**——Swap 在 SSD 上也很贵：等于员工不停跑仓库
4. **卸载 ≠ 清空**——人走了，隔间档案柜可能还在（`Library`）
5. **多 Agent 要隔离**——两人改同一份原件必打架；用 worktree / 容器复印工作区

## 电梯演讲（30 秒版）

> 操作系统是公司总管：给每个程序分前台工位和办公桌。内存是桌面，硬盘是仓库，桌面不够就开始跑仓库——这就是卡。进程是部门，线程是部门里的员工。读懂类比，你就能解释 Chrome 吃内存、Docker 在 Mac 上重、以及为什么要重启清场。

## 公司结构图（全书地图）

```
你（老板：下任务）
        │
        ▼
操作系统（行政总管）
  ├── 调度：谁坐前台工位（CPU）
  ├── 内存：桌面怎么分、满了怎么搬仓库
  ├── 文件：档案室怎么存怎么取
  ├── 网络：对外寄合同、对内对讲
  └── 安全：门禁、隔间、钥匙串
        │
        ▼
硬件（公司资产）
  CPU 工位 · RAM 桌面 · SSD 仓库 · 屏幕 / 网卡
```
