# 第 17 课 · PSBT 与多重签名

> BIP-174：部分签名比特币交易，多签与硬件钱包的标准格式。
>
> **作者：曹宇春**

---

## 为什么需要 PSBT

| 场景 | 难点 | PSBT 解决 |
|------|------|-----------|
| 冷/热分离 | 离线机无 UTXO 信息 | 携带 prevtx 数据 |
| 多签 | 多方分别签名 | 增量签名状态 |
| 硬件钱包 | 主机不可信 | 设备只签展示过的摘要 |

**PSBT** = 可传输、可部分签名的交易表示（Base64 或二进制）。

---

## PSBT 角色

```
Creator → Updater → Signer(s) → Finalizer → Extractor → Broadcast
```

- **Creator：** 建空白 PSBT
- **Updater：** 加 input/output、UTXO 详情
- **Signer：** 各签各的 input
- **Finalizer：** 合成 witness/scriptSig
- **Extractor：** 导出裸 raw tx

---

## 2-of-3 多签概念

```bash
# Bitcoin Core 创建 descriptor 钱包（简化示例）
bitcoin-cli -regtest createwallet "multisig" true true "" true true

# 使用 sortedmulti descriptor
# wsh(sortedmulti(2, key1, key2, key3))
```

链上 `scriptPubKey` 含 `OP_2 <pk1> <pk2> <pk3> OP_3 OP_CHECKMULTISIG`。

花费需任意 2 方签名。裸多签**费大、隐私差**；新项目考虑 **Taproot + MuSig2**。

---

## PSBT 工作流（CLI 概念）

```bash
# 1. 创建 PSBT
bitcoin-cli -regtest walletcreatefundedpsbt \
  '[]' '[{"<addr>":0.01}]'

# 2. 签名
bitcoin-cli -regtest walletprocesspsbt <psbt>

# 3. 完成并广播
bitcoin-cli -regtest finalizepsbt <psbt>
bitcoin-cli -regtest sendrawtransaction <hex>
```

多钱包场景用 `joinpsbts` 合并签名。

---

## 硬件钱包流程

1. 主机构造 PSBT，只含必要 UTXO 信息
2. USB/NFC 连硬件，屏幕显示金额/地址
3. 用户确认，设备签名
4. 主机 finalize + broadcast

**绝不**把助记词输入主机。

---

## 本课小结

- PSBT = 多签/离线的工业标准
- 2-of-3 适合公司金库、继承规划
- 优先 Taproot MuSig 而非 legacy multisig
