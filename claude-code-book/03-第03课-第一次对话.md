# 第03课 · 第一次对话

## 本课目标

- 完成第一个真实小任务
- 学会验收它的修改

## 3.1 入门任务模板

```
【任务】具体要做什么
【范围】只动哪些文件
【参考】参考哪个写法
【约束】不要做什么
【验证】改完后跑什么命令
```

## 3.2 示例：写 README

```
在当前项目创建 README.md，包含：
1. 项目简介（一句话）
2. 安装步骤
3. 一个使用示例
只创建 README.md，不要动其他文件。
```

## 3.3 验收四步

1. **看 diff** — 改了哪些文件？
2. **读逻辑** — 改动合理吗？
3. **跑验证** — 测试/构建通过吗？
4. **决定** — 接受 / 让它改 / 自己改

## 3.4 控制语句

| 你说 | 效果 |
|------|------|
| `先别改，只分析` | 只读 |
| `只做第 1 步` | 小步迭代 |
| `范围太大，只改 xxx` | 收窄范围 |

## 本课练习

在练习目录完成任务 A（写 README），并回答：它创建了几个文件？有没有不满意的地方？

## 本课小结

> 第一次对话的目标是走完「下任务 → 看结果 → 验收」全流程。
