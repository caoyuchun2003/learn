# 第06课 · 工作模式

## 本课目标

- 掌握 Plan → 审计划 → 执行 → 验证流程
- 知道何时只读、何时动手

## 6.1 三种模式

| 模式 | 何时用 |
|------|--------|
| 只读 / Ask | 读代码、出方案、review |
| Plan | 大功能先出计划 |
| Agent / 默认 | 确认后动手实现 |

## 6.2 大任务推荐流程

```
1. 「先读代码，出实现计划，不要改文件」
2. 你审计划，砍掉不必要的
3. 「按计划从第 2 步开始实现」
4. 「跑测试验证」
```

## 6.3 示例

**差：**「帮我加用户登录」

**好：**
```
先分析现有 auth 模块，给出登录实现计划（3-5 步），
列出要改的文件，不要写代码。
```

## 本课练习

给一个「加搜索功能」需求，只要计划，不要代码。

## 本课小结

> 大任务先 Plan，小任务直接干。计划是你和 AI 的契约。
