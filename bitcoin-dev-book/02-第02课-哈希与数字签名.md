# 第 02 课 · 哈希与数字签名

> 比特币用到的密码学 primitives，够用即可，不必推导证明。
>
> **作者：曹宇春**

---

## SHA-256：区块与交易的指纹

**性质（开发需要知道的）：**

- 固定长度输出（256 bit）
- 单向：已知 x 易算 hash(x)，已知 hash 难反推 x
- 抗碰撞：难找 x ≠ y 使得 hash(x) = hash(y)
- **雪崩效应**：输入改 1 bit，输出面目全非

**在比特币中的用途：**

| 用途 | 说明 |
|------|------|
| 区块哈希 | `hash(block_header)` 必须 < target |
| txid（legacy） | 对序列化交易 double SHA-256 |
| Merkle 树 | 叶子节点为交易哈希 |
| 地址 | `RIPEMD160(SHA256(pubkey))` |

```bash
# 命令行体验
echo -n "hello" | openssl dgst -sha256
```

**Double SHA-256：** 比特币中对交易、区块头常用 `SHA256(SHA256(x))`，记作 hash256。

---

## RIPEMD-160

公钥经 SHA-256 后再 RIPEMD-160，得到 20 字节，缩短地址长度。

```
pubkey (33/65 bytes)
    → SHA256 → 32 bytes
    → RIPEMD160 → 20 bytes (= pubkey hash)
```

---

## ECDSA 与 secp256k1

比特币使用 **椭圆曲线数字签名算法（ECDSA）**，曲线为 **secp256k1**（与以太坊相同曲线，格式细节不同）。

### 密钥对

- **私钥**：256 bit 随机数（严格说需在 [1, n-1] 范围内）
- **公钥**：私钥 × 基点 G（椭圆曲线点）
- **压缩公钥**：33 字节（0x02/0x03 + x 坐标）
- **非压缩公钥**：65 字节（0x04 + x + y）

### 签名

对交易摘要 `z` 用私钥签名，得到 `(r, s)`。  
验证方用公钥验证 `(r,s)` 与 `z` 是否匹配。

**Schnorr 签名**（BIP-340，Taproot 后）：更短、可批量验证、支持 MuSig 多签聚合。

### 交易中的意义

- **拥有私钥** = 能花费锁定到对应公钥/地址的 UTXO
- 签名放在 input 的 `scriptSig` 或 **witness**（SegWit）中

---

## 哈希指针 = 区块链接

区块头包含 `prev_block_hash`，即前一区块头的 hash256。

```
Block N-1  header  --hash-->  Block N header.prev_block_hash
```

改 Block N-1 的内容 → 其哈希变 → Block N 及之后全部无效（除非重做 PoW）。

## 动手：Python 生成密钥（理解用）

```python
# 需要 pip install ecdsa 或使用 bitcoinlib
# 仅演示概念，生产环境用成熟库
import hashlib
import os

privkey = os.urandom(32).hex()
print("private key (hex):", privkey)
```

生产环境请使用 **bitcoinlib**、**rust-bitcoin**、**bitcoinjs-lib**，不要自己实现密码学。

## 本课小结

| 算法 | 作用 |
|------|------|
| SHA-256 / hash256 | 区块 PoW、txid、Merkle |
| RIPEMD-160 | 压缩公钥 → 地址 |
| ECDSA secp256k1 | 交易授权 |
| Schnorr (BIP-340) | Taproot 默认签名 |
