# 附录 · 常见坑与对策

| 坑 | 表现 | 对策 |
|----|------|------|
| 一次塞太多 | 改一堆文件、引入新 bug | 拆任务，每轮一个目标 |
| 不跑测试 | 「好了」其实没过 | 要求改完跑 xxx 并贴结果 |
| 不看 diff | 悄悄改坏别的模块 | 大改必须逐文件审 |
| 对话太长 | 开始重复、变笨 | 新开对话 + CLAUDE.md |
| 用过时 API | 用了废弃写法 | 接 Context7 / 文档 MCP |
| 范围蔓延 | 顺便重构全项目 | 说「最小改动，只动 xxx」 |
| 盲信 review | AI 说没问题就 merge | 人审 + 跑 CI |
| 泄露密钥 | Token 贴进对话 | 用环境变量，不进 prompt |

## 救命句式

```
停。范围太大了，只改 src/auth/login.ts。
先别改，只分析。
回退上一个方案，换更简单的方式。
改完后运行 pnpm test，把完整输出贴出来。
```

## 心态

- AI 犯错是正常的，验收是你的责任
- 好 prompt 是迭代出来的，存成模板
- 先小步跑通，再加大难度
