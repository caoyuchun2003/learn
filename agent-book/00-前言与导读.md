# 📌 导读（全景总结）

## 一切的地基：三句话

> 所有「要不要上 Agent」的争论，都能追到这里。

```
① Agent = LLM + 工具 + 循环（能动手，所以又贵又险）
② 大多数产品不需要自主 Agent，需要 Workflow + RAG
③ 先问「任务步骤能预先定吗」，再问「要不要 ReAct」
```

## 本书分四层

```
决策层：该不该上 Agent · Workflow 怎么选 · RAG 够不够
模式层：提示链 · 路由 · 并行 · 编排 · 评估-优化 · 自主 Agent
工程层：工具 · MCP · 记忆 · Eval · 降级
场景层：教材阅读器 · 网校答疑 · 架构演进
```

## 💎 五条核心心法（只记这 5 句）

1. **能用 Workflow 别上 Agent**（步骤能写进代码就别交给 ReAct）
2. **能用 RAG 别直灌章节**（检索不到再考虑塞整节正文）
3. **Agent 能动手 = 必须人在环**（审批、沙箱、可撤销）
4. **先 V1 跑通再演进**（提示链 → 路由+RAG → 最后才自主 Agent）
5. **在你自己的任务上 Eval**（别因为看了 AgentWiki 就觉得万事都要 Agent）

## 🧭 三个决策口诀

```
上不上 Agent：  步骤固定→Workflow · 要先分类→路由 · 真没法预定义→Agent
上不上 RAG：    知识在库里→检索 · 单章短→可直灌 · 要强出处→必须 RAG
上不上工具：    只问答→不用 · 要查库/算数/调 API→Function Calling · 要多步探索→Agent 循环
```

## 🧠 总心法（一句话）

> Agent 不是「更聪明的聊天」；是 **把下一步交给模型决定的循环**。
> 所以它适合 **开放任务**，不适合 **你本来就能写死的流程**。
