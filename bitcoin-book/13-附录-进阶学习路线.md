# 附录 B · 进阶学习路线

> 本书只负责「讲给普通人听」。若你自己想从技术角度深入，请读 **《比特币技术全书》**（`../bitcoin-dev-book/`），或按下面路线自学。
>
> **作者：曹宇春**

---

## 适合谁

- 读完本书、想读比特币白皮书的人
- 有编程基础、想写链上程序的人
- 想区分「比特币」和「以太坊 / 炒币项目」的人

**爸妈和同事不需要读本篇。**

---

## 阶段一：协议理解（2–3 周）

### 必读

- [Bitcoin Whitepaper](https://bitcoin.org/bitcoin.pdf)（9 页，读 2 遍）
- 《精通比特币》第 1–7 章

### 要搞懂

- UTXO 模型（本书用现金条类比）
- 区块头、Merkle Tree、PoW
- 脚本、地址类型（P2PKH、SegWit、Taproot）

### 实践

- 安装 Sparrow Wallet，连接 testnet
- 在 [mempool.space](https://mempool.space) 解析一笔真实交易

---

## 阶段二：动手开发（3–4 周）

### 技术栈

```
Bitcoin Core (regtest) + bitcoin-cli
        或
rust-bitcoin / bitcoinjs-lib
```

### 练手项目

1. 解析一笔交易的 input / output
2. 在 regtest 上构造并广播签名交易
3. 简易 2-of-3 多签（PSBT 流程）

### 资源

- [learnmeabitcoin.com](https://learnmeabitcoin.com/)
- [bitcoin.org 开发者指南](https://bitcoin.org/en/developer-guide)

---

## 阶段三：安全与进阶（选读）

| 专题 | 资源 |
|------|------|
| 合约安全思维 | Ethernaut 前若干关（理解攻击思路） |
| 闪电网络 | Mastering the Lightning Network |

| 节点运维 | Bitcoin Core 全节点同步 testnet |

---

## 和以太坊路线的分叉

| | 比特币 | 以太坊 |
|--|--------|--------|
| 重点 | 交易、UTXO、节点 | 合约、DApp、Gas |
| 语言 | Script（有限） | Solidity |
| 先学谁 | 想懂「钱」先 BTC | 想写 DApp 先 ETH |

建议：**先比特币打牢 UTXO 和 PoW，再学以太坊。**
