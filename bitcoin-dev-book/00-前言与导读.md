## 📌 导读（全景总结）

### 一切的地基：三句话

> 23 课全长在这三个根上，所有「为什么」都能追到这里。

```
① 比特币账本 = UTXO 集合 + 最长（最多工作量）链上的区块历史
② 花费 = 用私钥解锁旧 UTXO（input），创建新 UTXO（output）
③ 安全 = 密码学签名 + PoW 让改历史成本远高于诚实挖矿
```

### 全书分四篇

```
基础篇：密码学、Merkle、UTXO（第 01–04 课）
协议篇：交易、Script、区块、PoW、分叉（第 05–09 课）
演进篇：SegWit、Taproot、P2P、全节点（第 10–14 课）
实战篇：regtest 开发、PSBT、安全、闪电（第 15–21 课）
```

### 技术栈一览

```
协议层：UTXO + Script + PoW
节点：Bitcoin Core（C++）
开发：bitcoin-cli / RPC、regtest、testnet
库：bitcoinjs-lib、rust-bitcoin、python-bitcoinlib
索引：mempool.space API、Esplora
二层：闪电网络（BOLT 协议）
```

### 五条核心心法

1. **先 UTXO，后账户**——比特币没有余额字段，只有未花费输出
2. **先验证，后信任**——全节点的意义是自己跑完所有规则
3. **Script 非图灵完备**——故意限制表达力，降低攻击面
4. **软分叉向前兼容，硬分叉分裂链**——升级逻辑要分清
5. **能在 regtest 上构造交易，才算入门开发**

## 环境准备

```bash
# macOS 安装 Bitcoin Core（含 bitcoind、bitcoin-cli）
brew install bitcoin

# 验证
bitcoind --version
bitcoin-cli --version
```

开发阶段优先用 **regtest**（本地私链），不要直接在 mainnet 试错。
