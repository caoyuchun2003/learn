# 第07课 · 权限与安全

## 本课目标

- 建立安全的协作习惯
- 知道什么该拒绝、什么不能给 AI

## 7.1 需要警惕的操作

- 删除文件 / 目录
- `git push --force`
- 改 `.env`、生产配置
- 安装未知依赖
- 跑不明 shell 脚本

## 7.2 安全习惯

| 习惯 | 原因 |
|------|------|
| 改前 `git status` | 知道起点 |
| 大改前开分支 | 可回滚 |
| 不贴密钥/Token | 防泄露 |
| 人审 diff 再 merge | AI 会犯错 |

## 7.3 推荐话术

```
不要改 .env 和配置文件。
commit 前给我看 message，我确认后再提交。
不要 force push。
```

## 本课练习

故意给一个会动很多文件的任务，练习喊停和收窄范围。

## 本课小结

> AI 是实习生权限：能干活，但重要操作要你点头。
