# 第07课 · 场景作战卡：加小功能

## 本课目标

- 小功能一次说清范围
- 用参考实现减少跑偏

## Skill

- 参考现有写法，不发明新风格
- **最小改动**，不顺便重构
- 大功能先 Plan 再实现

## Context

| 必给 | 说明 |
|------|------|
| 功能描述 | 输入、输出、边界 |
| 参考文件 | `@` 类似 Controller/组件 |
| 数据模型 | 表结构或类型定义 |
| 约束 | 只改哪些文件 |

## Feedback

- 新功能有测试（或你认可的手动验证步骤）
- 相关测试全过

## 开口模板

```
在 CourseController 加课程搜索：title 模糊搜、每页 10 条。
参考 @StudentController.php 的 index 写法。
复用现有 API 响应格式。只改相关文件，加参数验证。
```

## 本课练习

用模板描述你下一个要做的功能，检查 4 项 Context 是否齐全。

## 本课小结

> 加功能 = 说清「要什么」+ 指对「参考谁」。
