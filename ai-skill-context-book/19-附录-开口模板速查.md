## 开发类

### 修 Bug
```
@（文件）
报错：（粘贴）
复现：（命令）
先列 2-3 假设，最小修复，跑测试贴结果。
```

### 加功能
```
在（位置）加（功能描述）。
参考 @（文件）。只改相关文件。
（约束）。做完跑（测试命令）。
```

### 读代码
```
先别改代码。读 @（目录/文件）和 CLAUDE.md，
说明架构和 3 个核心模块。
```

### Review
```
Review 相对 main 的改动，按严重程度高→低列问题，先别改代码。
```

### Commit
```
根据 staged 写 conventional commits message，我确认后再 commit。
```

### 重构
```
重构 @（文件）：（目标）。API 不变。每步跑（测试）。不要动其他文件。
```

### 查 API
```
用（框架）（版本）做（任务）。先查最新文档再写代码，不要用过时写法。
```

## 产品类

### 新想法
```
【产品】【用户】【场景】【阶段】【材料】
按分析结构 1-7 输出，假设≥5 条带验证方法，不编造数据。
```

### 竞品
```
【我们】【竞品 A/B/C】【目的】
对比表 + 跟进/差异化/放弃 各 3 条。
```

### 排期
```
【候选功能】【约束】
RICE 排序：TOP2、可延后、不做。
```

### 分析 → 开发
```
【PRD 摘要】【验收标准】【不做清单】
按 MVP 第 1 项实现，最小改动，跑测试。
```

## 通用控制句

```
先别改，只分析。
只做第 N 步，做完停。
范围太大，只改 xxx。
改完跑 xxx，把完整输出贴出来。
不清楚的先问我，别假设。
```
