# 第18课 · 迭代式开发

## 本课目标

- 了解 Ralph 迭代技法
- 知道何时适合让 AI 多轮自推进

## 18.1 什么是迭代式开发？

给 AI 一个 **明确完成标准**，让它多轮循环：

```
写代码 → 跑测试 → 失败则修 → 再跑 → 直到通过
```

## 18.2 官方参考

插件 `ralph-wiggum`（`code/cyc/claude-code/plugins/ralph-wiggum/`）实现了这种自循环思路。

## 18.3 适用条件

**✅ 适合：**
- 完成标准可自动验证（测试全绿、lint 过）
- 目标清晰、范围有限

**❌ 不适合：**
- 需求模糊
- 没有自动化验证
- 涉及产品/UI 主观判断

## 18.4 关键：完成标准

```
完成标准：pnpm test 全部通过，且 coverage 不低于 80%。
最多迭代 10 轮，每轮汇报改了什么。
```

## 本课练习

为一个有单元测试的小函数，写下可自动验证的「完成标准」。

## 本课小结

> 迭代式开发 = 自动验证 + 明确终点。没有终点就会无限循环。
