# 第 10 课 · 地址类型演进

> 从 P2PKH 到 Bech32 再到 Taproot，编码的是 script 承诺。
>
> **作者：曹宇春**

---

## 地址不是账户

**地址**是 `scriptPubKey` 的人类可读编码，方便转账时指定锁定条件。

链上不存在「地址余额」，只有 UTXO 的 `scriptPubKey` 匹配。

---

## 演进时间线

```
2009  P2PKH      1...
2012  P2SH       3...
2017  SegWit     bc1q... (P2WPKH / P2WSH)
2021  Taproot    bc1p... (P2TR)
```

---

## P2PKH（Pay to Public Key Hash）

```
scriptPubKey: OP_DUP OP_HASH160 <20-byte> OP_EQUALVERIFY OP_CHECKSIG
地址: Base58Check, 版本字节 0x00 → 主网以 1 开头
```

---

## P2SH（Pay to Script Hash）

把**复杂赎回脚本**先 HASH160，链上只暴露脚本哈希。

```
scriptPubKey: OP_HASH160 <20-byte scriptHash> OP_EQUAL
赎回时 scriptSig: <sig> <redeemScript>
```

用例：多签、嵌套 SegWit（P2SH-P2WPKH）。

---

## Bech32 / Bech32m 编码

SegWit 与 Taproot 使用 **Bech32**（SegWit）或 **Bech32m**（Taproot v1+）：

- 小写 `bc1` 主网前缀
- 内置校验和，防输错
- 无大小写混用歧义（Bech32 限制）

---

## 生成地址（概念）

```text
私钥 → 公钥 → HASH160(pubkey) → Base58Check → P2PKH 地址
私钥 → 公钥 → Witness program → Bech32 encode → bc1q...
```

使用库完成，不要手写 Base58。

---

## 本课小结

- 地址 = scriptPubKey 的编码，不是链上原生概念
- P2PKH / P2SH / P2WPKH / P2TR 对应不同锁定逻辑
- 新钱包应默认 **bc1q** 或 **bc1p**
