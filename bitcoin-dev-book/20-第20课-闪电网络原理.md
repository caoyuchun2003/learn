# 第 20 课 · 闪电网络原理

> 链下通道、HTLC、路由——比特币二层支付。
>
> **作者：曹宇春**

---

## 解决什么问题

链上比特币：

- ~10 分钟确认
- 主网 fee 波动
- 隐私性有限

**闪电网络（LN）** 在链下建立支付通道，即时、小额、低费；最终仍锚定比特币主链结算。

---

## 支付通道（Payment Channel）

两人各锁一笔资金到 **2-of-2 多签** UTXO：

```
链上: Funding TX → 2-of-2 output
链下: 不断更新「最新余额分配」签名状态
关闭: 合作关闭（一笔 tx）或单方强制关闭（带延迟惩罚）
```

**关键：** 最新状态签名可链上执行；广播旧状态会被 **penalty** 罚没保证金。

---

## HTLC（Hash Time Locked Contract）

跨通道路由的核心：

```
「若你知道 preimage R，使得 hash(R)=H，则在时间 T 前你可领取；
 否则收款方超时收回。」
```

多跳路径上形成 **哈希锁链**，原子性：要么全成功，要么全失败。

---

## 路由

A → B → C → D 付款：

- A 不必信任 B、C，密码学 + 经济激励保证
- 每跳收取微小 routing fee
- 路径发现：gossip 网络广播 channel graph

## 限制与风险

- 需**在线**收 HTLC（接收方）
- **入金流动性**：通道 inbound capacity
- 通道对手方可 **force close**
- 实现漏洞历史上有真实损失（需跟版本）

---

## 本课小结

- LN = 链下状态更新 + 链上锚定 + HTLC 路由
- 适合高频小额；大额仍常用链上多确认
- 深入读：[Mastering the Lightning Network](https://github.com/lnbook/lnbook)
