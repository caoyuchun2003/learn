# 第 13 课 · P2P 网络与节点类型

> 交易和区块如何在互联网上传播。
>
> **作者：曹宇春**

---

## 网络拓扑

比特币是 **无中心 P2P 网络**：

- 节点间 TCP 连接（默认 8333 mainnet）
- 无全局目录服务器，靠 **DNS seed**、硬编码种子节点发现 peer
- 消息二进制协议，`magic bytes` 区分 mainnet/testnet

---

## 主要消息类型

| 消息 | 作用 |
|------|------|
| `version` / `verack` | 握手、协商版本 |
| `inv` | 清单：有新 tx/block 的哈希 |
| `getdata` | 请求具体 tx/block |
| `tx` | 传播交易 |
| `block` | 传播区块 |
| `headers` | 传播区块头（轻同步） |
| `addr` | peer 地址 gossip |

**传播模型：** 新交易 → inv → getdata → tx；节点验证后入 mempool，继续转发。

---

## 节点类型

| 类型 | 特点 |
|------|------|
| **全节点** | 完整验证，转发区块/交易 |
| **剪枝节点** | 删旧块数据，保留近期 + UTXO |
| **仅网络中继** | 部分节点限制 RPC、不服务轻钱包 |
| **SPV 客户端** | 只拉 headers + Merkle proof |
| **挖矿节点** | 全节点 + 矿池 Stratum 连接 |

---

## Mempool（内存池）

每个全节点本地维护未确认交易集合：

- 按 **ancestor fee rate** 等排序供矿工选题
- **容量限制** → 低费交易被驱逐
- **冲突检测** → 双花 input 拒绝

```bash
bitcoin-cli getmempoolinfo
bitcoin-cli getrawmempool true
```

---

## 隐私与网络层

- P2P 流量未默认加密（v2 transport BIP-324 逐步推广加密）
- 链分析可通过 **节点连接模式、地址重用** 推断身份
- 运行自己的全节点可改善隐私（不泄露查询地址给第三方 API）

---

## 本课小结

- 比特币靠 P2P gossip 传播 tx/block
- 全节点 = 验证 + 中继 + mempool
- 开发可依赖自建节点 RPC，而非只信区块浏览器 API
