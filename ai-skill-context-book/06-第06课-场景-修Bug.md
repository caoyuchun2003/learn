# 第06课 · 场景作战卡：修 Bug

## 本课目标

- 用 Skill + Context 结构化调试
- 避免 AI 瞎改一通

## Skill

- 调试四步：**现象 → 假设 → 验证 → 最小修复**
- 先分析再动手
- 改完必须跑测试

## Context

| 必给 | 说明 |
|------|------|
| 完整报错栈 | 不要只写「报错了」 |
| 复现步骤 | 运行哪条命令 |
| 相关文件 | `@` 1–3 个源文件 |
| 你已尝试的 | 避免重复劳动 |
| 测试命令 | 来自 CLAUDE.md |

## Feedback

- 原失败用例变绿
- `pnpm test`（或项目等价命令）输出贴回

## 开口模板

```
@src/xxx.ts
报错：（完整粘贴）
复现：pnpm test xxx
先列 2-3 个假设，用最小改动修复，跑测试贴结果。
```

## 本课练习

拿一个真实 bug 走完整流程，记录 Context 缺了啥时 AI 变蠢。

## 本课小结

> 修 bug 的 Context 核心是完整报错 + 复现路径。
