# 第 09 课 · Demo2：园区接入与 Trunk

> 多台 S5130 接入 + 上联 Trunk 到核心——典型园区接入层。
>
> **作者：曹宇春**

---

## 场景故事

一栋楼 **3 层接入交换机**，分别接办公、会议、监控；**上联一条 Trunk** 到核心（本 Demo 用一台 **Core-SW** 模拟核心，也可实机 S7500）。

---

## 拓扑

```
              [ Core-SW ]
           Vlan-int 10/20/30
           GE1/0/49 Trunk
        ┌──────┼──────┐
    [SW-F1]  [SW-F2]  [SW-F3]
    acc 10   acc 20   acc 30
    办公     会议     监控
```

---

## IP 规划

| VLAN | 用途 | 网关（Core） |
|------|------|--------------|
| 10 | Office | 192.168.10.1/24 |
| 20 | Meeting | 192.168.20.1/24 |
| 30 | Camera | 192.168.30.1/24 |
| 99 | 管理 | 192.168.99.1/24 |

互联：Core `192.168.99.1`，接入交换机管理 IP 在 VLAN 99。

---

## 一、Core-SW（核心/汇聚）

```text
<H3C> system-view
[H3C] sysname Core-SW

[Core-SW] vlan 10
[Core-SW-vlan10] quit
[Core-SW] vlan 20
[Core-SW-vlan20] quit
[Core-SW] vlan 30
[Core-SW-vlan30] quit
[Core-SW] vlan 99
[Core-SW-vlan99] name Mgmt
[Core-SW-vlan99] quit

# Trunk 下联 F1（示例一口，其他口类似）
[Core-SW] interface GigabitEthernet1/0/1
[Core-SW-GigabitEthernet1/0/1] description Downlink-to-SW-F1
[Core-SW-GigabitEthernet1/0/1] port link-type trunk
[Core-SW-GigabitEthernet1/0/1] port trunk permit vlan 10 20 30 99
[Core-SW-GigabitEthernet1/0/1] undo shutdown
[Core-SW-GigabitEthernet1/0/1] quit

# 三层网关
[Core-SW] interface Vlan-interface 10
[Core-SW-Vlan-interface10] ip address 192.168.10.1 255.255.255.0
[Core-SW-Vlan-interface10] quit
[Core-SW] interface Vlan-interface 20
[Core-SW-Vlan-interface20] ip address 192.168.20.1 255.255.255.0
[Core-SW-Vlan-interface20] quit
[Core-SW] interface Vlan-interface 30
[Core-SW-Vlan-interface30] ip address 192.168.30.1 255.255.255.0
[Core-SW-Vlan-interface30] quit
[Core-SW] interface Vlan-interface 99
[Core-SW-Vlan-interface99] ip address 192.168.99.1 255.255.255.0
[Core-SW-Vlan-interface99] quit

[Core-SW] save
```

---

## 二、SW-F1（接入，办公 VLAN10）

```text
[H3C] sysname SW-F1

[SW-F1] vlan 10
[SW-F1-vlan10] quit
[SW-F1] vlan 99
[SW-F1-vlan99] quit

# 上联 Trunk
[SW-F1] interface GigabitEthernet1/0/48
[SW-F1-GigabitEthernet1/0/48] port link-type trunk
[SW-F1-GigabitEthernet1/0/48] port trunk permit vlan 10 99
[SW-F1-GigabitEthernet1/0/48] undo shutdown
[SW-F1-GigabitEthernet1/0/48] quit

# 办公口
[SW-F1] interface GigabitEthernet1/0/1
[SW-F1-GigabitEthernet1/0/1] port link-type access
[SW-F1-GigabitEthernet1/0/1] port access vlan 10
[SW-F1-GigabitEthernet1/0/1] undo shutdown
[SW-F1-GigabitEthernet1/0/1] quit

# 管理 IP（可选）
[SW-F1] interface Vlan-interface 99
[SW-F1-Vlan-interface99] ip address 192.168.99.11 255.255.255.0
[SW-F1-Vlan-interface99] quit
[SW-F1] ip route-static 0.0.0.0 0 192.168.99.1

[SW-F1] save
```

**SW-F2**：access vlan 20，trunk permit 20 99  
**SW-F3**：access vlan 30，trunk permit 30 99，监控口可 `poe enable`

---

## 三、验收 checklist

```text
□ Core：dis vlan → 10/20/30 存在
□ Core：dis int b → Trunk 口 UP
□ SW-F1：dis vlan 10 → Gi1/0/1 在 VLAN10
□ PC 192.168.10.10/24 网关 192.168.10.1 → ping 通
□ PC 192.168.20.10 ping 192.168.10.10 → 跨 VLAN 通（无 ACL 时）
□ SW-F1：ping 192.168.99.1 → 管理可达
```

---

## 四、Trunk 要点

| 要点 | 说明 |
|------|------|
| 两端都要 trunk | 一端 access 一端 trunk 会不通 |
| permit vlan | 没 permit 的 VLAN 过不了 Trunk |
|  native vlan | 注意两端一致（若用默认） |
| 网关只在 Core | 接入层通常 **不做 SVI**（本 Demo 仅管理 IP 在 99） |

---

## 五、扩展

- 核心换 **S7500 IRF 双机**：网关仍在 Core，接入配置不变  
- 接 **WAC**：Trunk 增加无线 AP 所在 VLAN  
- 出口接 **MSR**：Core 上 `ip route-static 0.0.0.0 0 x.x.x.x` 指 MSR

## 小结

- 园区接入 = access 口 + 上联 trunk  
- 三层网关在核心；接入仅二层（+ 可选管理 VLAN）
