# 第05课 · 作战卡总览

## 本课目标

- 掌握通用作战卡格式
- 知道 8 个场景课各解决什么

## 5.1 通用作战卡

每个场景固定三栏：

| 维度 | 问什么 |
|------|--------|
| **Skill** | 标准流程、规范、禁忌 |
| **Context** | 必须喂的文件和信息 |
| **Feedback** | 怎么算做完 |

每次任务前：

```
□ Skill 齐了？
□ Context 齐了？
□ Feedback 说清了吗？
```

## 5.2 八大战场（第06–13课）

| 课 | 场景 |
|----|------|
| 06 | 修 Bug |
| 07 | 加小功能 |
| 08 | 读代码 / Onboarding |
| 09 | Code Review |
| 10 | Commit / PR |
| 11 | 重构 |
| 12 | 文档 / 知识库 |
| 13 | 查最新 API |

## 5.3 建议攒的个人 Skill 库（先 5 个）

1. 调试四步
2. 小功能：参考现有 + 最小改动
3. Review 四维度
4. 安全红线（.env、force push）
5. 项目规范（来自 CLAUDE.md）

## 本课练习

从上面 8 个场景选你最常干的 2 个，跳到对应课，用今天真实任务练一次。

## 本课小结

> 作战卡 = 把经验变成 checklist，不用每次从零想 prompt。
