# 第15课 · 产品分析：框架与模板

## 本课目标

- 掌握固定输出结构
- 会选分析框架

## 15.1 固定输出结构（Skill）

每份分析建议包含：

```
1. 问题定义（一句话）
2. 目标用户 & 核心场景
3. 痛点 & 现有替代方案
4. 方案建议 & 差异化
5. 关键假设 + 验证方法
6. 风险 & 不做清单
7. MVP 建议 & 下一步
```

## 15.2 框架选型

| 框架 | 何时用 |
|------|--------|
| 用户-场景-痛点-方案 | 新想法、新功能 |
| 5W1H | 想法还很模糊 |
| 价值主张画布 | 定定位 |
| 假设-验证 | 防自嗨 |

## 15.3 角色 Skill

```
你是资深产品经理，有技术背景。
结论区分事实、推断、假设。
不清楚的先列问题，不要编造数据。
```

## 15.4 Context 必给

| 项 | 示例 |
|----|------|
| 产品一句话 | 帮老师 AI 批改作文 |
| 目标用户 | 初中语文老师 |
| 场景 | 课后批 30 篇 |
| 阶段 | 想法 / MVP / 已上线 |
| 你的目标 | 要不要做、先做哪 3 功能 |

## 开口模板 A：新想法

```
【角色】资深产品经理，区分事实与假设
【产品】（一句话）
【用户】【场景】【阶段】
【材料】（用户原话、竞品名；无则写「无」）

按固定输出结构 1-7 写。假设至少 5 条，每条带验证方法。
不要编造市场数据。
```

## 本课练习

用模板 A 分析一个你熟悉的小产品（可虚构），检查 7 节是否齐全。

## 本课小结

> 产品分析的 Skill 首先是「固定结构」，其次是「不编造」。
