# 第 03 课 · DeepSeek、文心、闭源怎么选

> 结论：**同一平台（如千帆）上，用「任务档 × 开通情况 × 单价」选型，不要死磕品牌情怀。**

---

## 选型三问

1. **这个 Key 到底开了哪些模型？**（列表接口 / 控制台，测一遍比猜强）  
2. **任务是导读短答，还是硬推理？**  
3. **挂了有没有降级模型？**

本站实测：旧模型名（如部分 `ernie-4.0-8k`）可能直接 `invalid_model`；**能列出的 chat 模型才是候选**。DeepSeek、文心新系列、GLM 等都会随账号能力变化——**以你账号列表为准**。

## 一张务实对照（2026 心智，非报价单）

| 偏好 | 更常落到 |
|------|----------|
| 高性价比短答 / 编程向 | DeepSeek 系列（如 `deepseek-v3.2`、`deepseek-v4-flash`） |
| 平台生态、中文产品文案 | 文心系列（以账号可点模型为准） |
| 极难推理 / 终审 | 更强档或人工；别默认给每个用户 |

**价格每天可能变**，本课不写死数字；打开计费页对照上面「简/中/难」。

## 选完还要验证的三件事

1. 同提示下答是否「手册未覆盖」过多（过严提示 + 弱模型会叠加）  
2. P95 延迟是否可接受  
3. 换模型后 JSON / 纯文本协议是否仍稳（你们流式用的是纯文本，非流式可走 JSON）

## 反模式

- 「网上说 XX 最强」就全切，不测自己的题集  
- 一个 Key 测通，生产换另一个账号权限不同  

## 练习

写 5 道自家黄金问题，在两个候选模型上盲测，记：可用 / 胡编 / 拒答过多。
