# 第 05 课 · 云骨干与 DCI 路由

> 多数据中心、多云——路由在「云网一体」里扮演什么角色？
>
> **作者：曹宇春**

---

## 先记住这个类比

- **单个 DC** = 一座 **立体仓库**（第 07 课 S 系列为主）  
- **DC 之间** = 仓库之间的 **高速公路 + 统一调度系统**  
- **CR/SR + EVPN** = 高速公路上的 **路牌与调度规则**  
- **S12500 VTEP** = 每个仓库门口的 **标准化装卸口**  

实践中 **数据面以 S 系列交换机（VXLAN VTEP）为主**，**CR/SR 作 BGP/EVPN 控制与广域路由**。

---

## 架构角色

```
  DC1 Leaf-Spine          DC2 Leaf-Spine
       │                        │
       └──── DCI 100G/400G ─────┘
                  │
            SR/CR（EVPN RR）
```

| 组件 | 典型产品 | 功能 |
|------|----------|------|
| DC 内 fabric | S6800、S12500 | VXLAN VTEP |
| DC 间链路 | 专线/波分 | 物理连通 |
| 广域控制 | CR、SR8800 | BGP EVPN、路由反射 |

---

## EVPN 为什么重要

**VXLAN** 解决租户隔离（VNI 1600 万级）；**EVPN** 用 BGP 传 MAC/IP，是 **控制平面**。

Type-2：VM 迁移 MAC/IP  
Type-5：跨 DC 的 IP 前缀  
Multi-homing：双活网关  

华三 **CloudOS + AD-NET + S 系列** 是私有云常见组合：云管开租户 → 控制器下发 → Comware 执行。

---

## AD-WAN 与云骨干的交汇

分支 **MSR（AD-WAN）** 访问多云时：

```
分支 ── Internet/MPLS ── 云骨干/DC ── 公有云
```

**策略统一**（哪些应用走本地 DC、哪些直连云）要 **AD-WAN 控制器 + 核心 SR** 协同——新华三「同厂云+网」的卖点。

---

## 智算 DCI（简提）

AI 训练集群跨 pod/DC 时，**DC 内 RoCE** 为主（第 07 课），DC 间 **400G + EVPN**；路由层提供 **可达性**，不替代 leaf 级时延优化。

---

## 在你项目里什么时候用得上

- **单 DC 私有云** → 第 07 课足够，本章可跳读  
- **双活 DC、VM 跨站迁移** → 必须理解 EVPN DCI  
- **分支上云** → MSR/AD-WAN + 本章 + 第 10 课

## 小结

- DCI = DC 间大带宽 + overlay 延伸  
- 数据面 S 系列 VTEP，控制面 SR/CR + EVPN  
- 与 CloudOS、AD-WAN 组成云网一体方案
