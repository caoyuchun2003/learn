# 第 05 课 · 交易结构与生命周期

> 交易的字段、序列化、txid 与 wtxid。
>
> **作者：曹宇春**

---

## 交易结构概览

```text
Transaction
├── version          (4 bytes)
├── marker + flag      (SegWit only: 0x00 0x01)
├── inputs[]           (vin)
│   ├── prev_txid
│   ├── prev_vout
│   ├── scriptSig / (empty for segwit)
│   └── sequence
├── outputs[]          (vout)
│   ├── value
│   └── scriptPubKey
└── locktime
    └── witness[]      (SegWit, per input)
```

---

## Input（vin）

| 字段 | 说明 |
|------|------|
| `prev_txid` | 引用的 UTXO 所在交易（小端序显示） |
| `prev_vout` | 输出索引 |
| `scriptSig` | 解锁脚本（legacy）或空（SegWit） |
| `sequence` | 默认 0xFFFFFFFF；用于 RBF、相对时间锁 |

**sequence 与 RBF：** `sequence < 0xFFFFFFFE` 表示该交易可被更高费版本替换（BIP-125）。

---

## Output（vout）

| 字段 | 说明 |
|------|------|
| `value` | satoshi 数量 |
| `scriptPubKey` | 锁定脚本，定义花费条件 |

---

## locktime

- `locktime = 0`：立即有效（仍须入块）
- `< 500000000`：区块高度锁定
- `≥ 500000000`：UNIX 时间戳锁定

配合 `OP_CHECKLOCKTIMEVERIFY`（CLTV）实现定时支付。

---

## txid vs wtxid

| | 计算方式 | 用途 |
|--|----------|------|
| **txid** | 对不含 witness 的序列化做 hash256 | 区块内 Merkle 树、OutPoint 引用 |
| **wtxid** | 对含 witness 的完整序列化做 hash256 | SegWit 后 peer 传播、防 witness 篡改 |

SegWit 引入 **tx witness** 与交易本体分离，修复**交易延展性**（第三方可改签名不改 txid）。

---

## 生命周期

```
constructed → signed → broadcast → mempool → mined → confirmed
                  ↓
              rejected (invalid / too low fee / orphan)
```

**孤儿交易（orphan）：** input 引用的 UTXO 尚不可见（父交易未确认），暂存 orphan pool。

---

## bitcoin-cli 查看交易

```bash
# testnet 示例
bitcoin-cli -testnet getrawtransaction <txid> true
bitcoin-cli -testnet decoderawtransaction <hex>
```

`verbose=true` 返回 JSON，含 `vin`、`vout`、`confirmations`。

---

## 本课小结

- 交易 = inputs 消费 UTXO + outputs 创建 UTXO
- SegWit 分离 witness，产生 wtxid
- sequence / locktime 支持 RBF 与时间锁
