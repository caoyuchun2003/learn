# 第 10 课 · 网络管理与自动化

> iMC 管设备，AD-NET 管策略，AD-WAN 管分支链路。
>
> **作者：曹宇春**

---

## 先记住这个类比

| 产品 | 类比 |
|------|------|
| **iMC** | 交通 **监控大屏 + 报警中心** |
| **AD-NET** | **智能红绿灯与可变车道**（SDN 自动配） |
| **AD-WAN** | 分支 **多路线导航**（哪条快走哪条） |
| **Cloudnet** | 小店 **手机看监控**，够用就好 |
| **NETCONF/Ansible** | 给系统 **批量发工单** 的 API |

```
  业务系统 / CloudOS
         │ API
  AD-NET / AD-WAN 控制器
         │
       iMC
         │
  MSR / S / WA / 防火墙 ...
```

---

## iMC：默认网管

**干什么**：拓扑发现、告警、配置备份、性能监控、资产报表  

**管什么**：S 系列、MSR/SR、WA/AC、SecPath 等  

**部署**：单机（中小）/ 分布式（大型政企）

**日常流**：发现设备 → 备份配置 → 告警邮件 → 拓扑钻取排障

---

## AD-NET：SDN 与 DC 自动化

- VXLAN 租户网络 **自动开通**  
- **策略随行**（用户到哪策略跟到哪）  
- 与 **CloudOS** 联动私有云  

**和 iMC 分工**：iMC = 设备健康；AD-NET = 业务策略与 overlay 编排

---

## AD-WAN：分支 SD-WAN

与 **MSR** 配合：

| 传统 WAN | AD-WAN |
|----------|--------|
| 逐台配 VPN | 模板 + 控制器下发 |
| 链路断手工切 | 应用探测 + 自动选路 |
| 流量黑盒 | 应用/链路质量可视 |

---

## Cloudnet：SMB 云管

交换机 + AP + 网关 **上云**，App 管理。  
**边界**：大型园区、金融政务 **数据不出域** → 用本地 iMC，不用 Cloudnet。

---

## 可编程（现代运维）

| 接口 | 用途 |
|------|------|
| NETCONF/YANG | Ansible 批量配置 |
| RESTful API | 自研 OSS |
| Telemetry | 实时指标 streaming |

**DC 规模** 下手工 CLI 不可持续——自动化不是炫技，是必需。

---

## 选型一张表

| 客户 | 推荐 |
|------|------|
| SMB / 门店 | Cloudnet |
| 中型园区 | iMC |
| 多分支 | iMC + AD-WAN |
| 私有云 DC | iMC + AD-NET + CloudOS |

---

## 在你项目里什么时候用得上

- 标书要求 **集中网管、配置备份、拓扑** → iMC  
- 标书要求 **SD-WAN、应用选路** → AD-WAN + MSR  
- 标书要求 **云平台网络自动化** → AD-NET

## 小结

- iMC = FCAPS 基础网管  
- AD-NET = DC/园区 SDN 自动化  
- AD-WAN = 分支 SD-WAN + MSR  
- 规模越大，越要 NETCONF/Telemetry
