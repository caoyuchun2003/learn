# 附录 A · Agent 方案决策树

> **一页纸拍板:从「要不要上」到「用哪种模式」。**
> 把决策篇(01–03)和模式篇(04–06)的判断合成一张可照着走的树。

---

## 主决策树(从上往下走)

```
【第 0 步】这个 AI 功能,用户到底要什么?
│
├─ 只要「答」(解释、问答、生成文本)
│     │
│     └─【第 1 步】知识从哪来?
│           ├─ 就眼前一小段          → 直灌当前段落 + 单次 LLM
│           ├─ 在教材 / 知识库里       → RAG(切块 + 检索 + 出处)
│           └─ 多种问题类型混着来      → 路由 + RAG(各类型专属分支)
│
└─ 要「办」(查数据、改状态、调后台)
      │
      └─【第 2 步】步骤能预先写死吗?
            ├─ 顺序固定的多步生成       → 提示链 Chaining
            ├─ 要先分类再分流           → 路由 Routing
            ├─ 独立子任务、想省时间      → 并行 Parallelization
            ├─ 子任务数量因输入而变      → 编排-执行 Orchestrator-Workers
            ├─ 同一产物要反复打磨        → 评估-优化 Evaluator-Optimizer
            ├─ 只查 / 改一两下(顺序固定)→ 单次 Function Calling
            └─ 真的没法预定义步骤        → 自主 Agent(最后才选)
```

---

## 三个决策口诀(速记)

```
上不上 Agent:  步骤固定 → Workflow · 要先分类 → 路由 · 真没法预定义 → Agent
上不上 RAG:    知识在库里 → 检索 · 单章短 → 可直灌 · 要强出处 → 必须 RAG
上不上工具:    只问答 → 不用 · 查库 / 算数 / 调 API → Function Calling · 多步探索 → Agent 循环
```

---

## 自主 Agent 准入四问(四条全「是」才上)

```
① 任务开放,步骤无法预先定义?          是 / 否
② 要多步,且下一步依赖上一步真实结果?    是 / 否
③ 工具组合不确定(可能查 / 算 / 改)?    是 / 否
④ 低频但高价值,扛得住成本和延迟?        是 / 否
```

> **只要有一个「否」→ 回退 Workflow。** 四个全「是」,再进上线清单。

---

## 自主 Agent 上线清单(缺一不可)

```
□ 沙箱          工具在受限环境执行
□ 人工审批      写 / 删 / 发等高风险操作逐次确认
□ 审计日志      每步「想 / 做 / 看」全记录
□ 上限熔断      最大轮数 / 成本 / 超时
□ 上下文管理    压缩旧工具结果,关键结论存记忆
□ Eval          在自己任务上评过成功率与轨迹
```

---

## 模式速查(六选一)

| 模式 | 一句话 | 属于 |
|------|--------|------|
| 提示链 Chaining | 固定几步串联 | Workflow |
| 路由 Routing | 先分类再分流 | Workflow |
| 并行 Parallelization | 独立子任务同时跑 | Workflow |
| 编排-执行 Orchestrator-Workers | 主脑动态拆子任务 | Workflow |
| 评估-优化 Evaluator-Optimizer | 生成↔评估循环打磨 | Workflow |
| 自主 Agent | 模型自己规划执行循环 | Agent |

> 从上到下越复杂、越贵、越难控。**永远选能解决问题的最简单那个。**
