# 第 11 课 · 按现象排障 CLI

> 口 Down、ping 不通、上不了网、VPN 不起、AP 不上线。
>
> **作者：曹宇春**

---

## 排障总流程

```
1. dis int b          → 物理/协议 Up？
2. dis ip int b       → IP 对吗？
3. dis ip routing-table → 路由对吗？
4. ping / tracert     → 断在哪一跳？
5. dis cur | dis logbuffer → 配置/日志
```

---

## 现象 → 命令对照表

| 现象 | 优先命令 | 常见原因 |
|------|----------|----------|
| 口 Down | `dis int Gi x/x/x` | shutdown、线缆、对端 Down |
| PC ping 不通网关 | `dis vlan`、`dis int b` | access vlan 错、网关 IP 错 |
| 网关通，跨 VLAN 不通 | `dis ip routing-table`、acl | 缺路由、ACL deny |
| 内网通，上不了网 | `dis ip rout`、`dis nat outbound` | 无默认路由、NAT 未配、ACL 网段错 |
| VPN 不起 | `dis ike sa`、`dis ipsec sa` | 密钥/ACL/对端地址错 |
| OSPF 邻居不起 | `dis ospf peer`、`dis int` | area 不一致、hello 被 ACL 挡 |
| AP 不上线 | `dis wlan ap all`、`dis poe int` | PoE 关、VLAN/DHCP、CAPWAP 不通 |
| 改完重启配置没了 | — | **忘了 save** |

## 案例 1：PC ping 不通网关

```text
□ PC 网关是否 192.168.10.1？
□ dis vlan 10 → PC 口是否在 VLAN10？
□ dis int b → 接入口 UP？
□ Core/SW dis int Vlan-interface 10 → UP？IP 对？
□ dis arp → 有没有 PC 的 ARP？
```

---

## 案例 2：能 ping 网关，上不了网

**分支（MSR）**

```text
□ MSR ping 202.1.1.1（运营商）通吗？
□ dis ip routing-table → 0.0.0.0/0 存在吗？
□ dis nat outbound → WAN 口绑 NAT 了吗？
□ acl 2000 是否包含 PC 网段？
□ dis nat session → 有转换条目吗？
```

**交换**

```text
□ dis ip routing-table → 0.0.0.0/0 指到 MSR 了吗？
```

---

## 案例 3：Trunk 不通

```text
□ 两端都是 trunk 吗？
□ port trunk permit vlan 包含业务 VLAN 吗？
□ 一端 access 一端 trunk → 常见错误
□ dis int b → 物理 UP 吗？
```

---

## 案例 4：IPSec VPN

```text
□ dis ike sa → 第一阶段 UP？
□ dis ipsec sa → 第二阶段 UP？
□ acl 感兴趣流是否覆盖 ping 的源目？
□ 对端 pre-shared-key 一致吗？
□ NAT 是否改写了 UDP 500/4500？（NAT-T）
```

## debugging 警告

```text
debugging ip packet   ← 生产慎用，流量大时可能拖死 CPU
undo debugging all    ← 用完必须关
```

优先 **display**，不得已再 debug。
