# AI 协作方法论 · 总导读

> 用 Skill 和 Context 驾驭 AI，从写代码到产品分析  
> 作者：曹宇春

---

## 📕 内容简介

同样一个问题，AI 有时很神、有时很蠢——差别往往不在模型，而在你有没有给它**对的 Skill（会什么）**和**对的 Context（看见什么）**。本书不谈某个具体工具按钮，而是回答一个更持久的问题：**怎么和 AI 协作才有效。**

全书 **17 课 + 2 篇附录 + 1 个实战案例**，分三篇：方法论（Skill、Context、完整公式）、场景作战卡（修 bug、加功能、Review、文档等 8 类高频任务）、产品分析（框架、竞品、排期、与开发衔接）。附录提供个人 Skill 库模板与开口模板速查；第 20 课为虚构教学案例「智练题库」，完整演示产品分析写法。

适合**不限于 Claude Code**——用 Cursor、ChatGPT 或任何 AI 工具的人都能用。读完你有一套可复用的协作框架，知道每种场景该装什么能力、喂什么信息、怎么验收。

## 👤 适合谁读

- 日常用 AI 写代码、但**效果不稳定**的开发者
- 想分清 Skill、Command、CLAUDE.md、MCP 关系的技术人
- 需要兼顾 **开发 + 产品分析** 的一人多岗角色
- 想把 prompt 经验沉淀成 **团队规范** 的人
- 已读或并行阅读《Claude Code 学习方法》的读者

## 🧭 怎么读

- **建立框架** → 先读第 01–05 课（方法论）
- **对着干活** → 按当天任务跳第 06–13 课作战卡
- **做产品** → 读第 14–17 课，并对照第 20 课案例自己写一版
- **速查** → 用附录 19 开口模板，复制改括号即用
- **与工具课配合** → 本课管「怎么协作」，《Claude Code 学习方法》管「工具怎么点」

## 一句话

```
AI 使用效果 ≈ Skill（会什么）× Context（看见什么）
```

## 本课程解决什么？

- 为什么同样的问题，有时 AI 很神、有时很蠢？
- Skill、Slash Command、CLAUDE.md 到底是什么关系？
- 修 bug、写功能、做 review、做产品分析，各该怎么喂 AI？
