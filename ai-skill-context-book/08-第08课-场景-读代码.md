# 第08课 · 场景作战卡：读代码与 Onboarding

## 本课目标

- 用只读模式快速摸清项目
- 新人上手或接陌生代码库

## Skill

- **只读**：先别改文件
- 先结构后细节
- 输出：入口、模块、数据流

## Context

- 目录结构（或让 AI `ls`/`tree`）
- 入口文件 `@`（main、index、routes）
- `CLAUDE.md`、README

## Feedback

能回答三问：
1. 入口在哪？
2. 改功能 X 该动哪些模块？
3. 测试怎么跑？

## 开口模板

```
先别改代码。读 @src/ 和 CLAUDE.md，
用 5 句话说明架构，列出 3 个核心模块及调用关系。
```

## 大项目三步

```
第1轮：列出跟「认证」相关的文件
第2轮：@auth/ 读文件，画调用关系
第3轮：（另开对话）开始改代码
```

## 本课练习

对一个陌生目录做只读分析，看输出能否当 onboarding 文档。

## 本课小结

> 读代码的 Skill 是「只读 + 结构化输出」。
